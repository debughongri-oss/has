  <name>Task 1: 创建 shared/auth.js 验证模块</name>
  <files>cloudfunctions/shared/auth.js</files>
  <behavior>
    - requireArtist(wxContext): 当 OPENID === ARTIST_OPENID 时返回 true，否则返回 false
    - requireArtist(wxContext): 当 OPENID 不匹配时返回 { errCode: -1, errMsg: '无权限操作' } 错误对象
    - sanitizeProfileUpdate(data): 只保留白名单字段 (name, bio, experience, experience_years, service_area, specialties, avatar, contact_info)，过滤掉其他字段
    - sanitizeProfileUpdate(data): 白名单中不存在的字段（如 _id, _openid, created_at）被移除
  </behavior>
  <action>Per D-10, D-11: 创建 `cloudfunctions/shared/auth.js` 模块。

硬编码 ARTIST_OPENID（与 `miniprogram/utils/constants.js` 保持一致：`oDmtI48gecuF3n9OkEBJN91BJliI`）。

导出两个函数：
1. `requireArtist(wxContext)` — 比较 `wxContext.OPENID` 与 `ARTIST_OPENID`，匹配返回 `{ ok: true }`，不匹配返回 `{ ok: false, response: { errCode: -1, errMsg: '无权限操作' } }`
2. `sanitizeProfileUpdate(data)` — Per D-13: 使用白名单 `['name', 'bio', 'experience', 'experience_years', 'service_area', 'specialties', 'avatar', 'contact_info']` 过滤 data 对象，只保留白名单内字段，返回过滤后的对象。用 `Object.keys(data).filter(key => ALLOWED_FIELDS.includes(key)).reduce((acc, key) => { acc[key] = data[key]; return acc }, {})` 实现。

注意：云函数的 require 路径。`cloudfunctions/shared/` 目录与各云函数平级，works 引用路径为 `require('../shared/auth')`。但微信云开发默认不会将 shared 目录作为云函数部署（没有 index.js 入口），所以不会被误部署。

在 shared/auth.js 文件顶部添加注释说明此模块供云函数 require 引用，不是独立云函数。</action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Cloud Function | 任何微信用户可调用云函数，传入任意 event 参数 |
| Cloud Function → Database | 云函数直接操作数据库，无中间层 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | S (Spoofing) | All write actions | mitigate | requireArtist 检查 wxContext.OPENID vs ARTIST_OPENID — 服务端身份不可伪造 |
| T-06-02 | T (Tampering) | profile update | mitigate | sanitizeProfileUpdate 白名单过滤 — 客户端无法注入 _id/_openid/created_at 等字段 |
| T-06-03 | I (Info Disclosure) | Read actions | accept | 读操作（list/detail/get）公开设计 — 数据本就是公开展示内容，无敏感信息泄露风险 |
| T-06-04 | E (Elevation) | All write actions | mitigate | 非化妆师身份执行写操作返回 errCode: -1 — 阻止权限提升 |
