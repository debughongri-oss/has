## Task 1: authService 增加 ensureLogin + refreshUserInfo
**Requirement:** SEC-03, SEC-06
**Files:** `miniprogram/services/auth.js`

**Changes:**
1. 新增模块级 `_loginPromise = null`（幂等缓存）
2. 新增 `ensureLogin()`：
   ```javascript
   const ensureLogin = () => {
     if (_userInfo) return Promise.resolve(_userInfo)
     if (_loginPromise) return _loginPromise
     _loginPromise = silentLogin().finally(() => { _loginPromise = null })
     return _loginPromise
   }
   ```
   注意：finally 清理 `_loginPromise` 让失败后可重试，但 `_userInfo` 成功后保留
3. 新增 `refreshUserInfo()`：profile 更新后调用，强制重新拉 users 权威数据
   ```javascript
   const refreshUserInfo = async () => {
     try {
       const result = await callCloudFunction('login', { action: 'getUser' })
       if (result.errCode === 0) {
         _userInfo = result.data
         return _userInfo
       }
       throw new Error(result.errMsg)
     } catch (err) {
       console.error('刷新用户信息失败:', err)
       throw err
     }
   }
   ```
4. `isArtist()` / `isLoggedIn()` / `getUserInfo()` 保持同步签名（避免大面积重构），但语义变为"必须在 ensureLogin 后调用才有意义"
5. 移除 `ARTIST_OPENID` import（Plan 01 后服务端权威，客户端 isArtist 改为读 `_userInfo.role === 'artist'` 或保留 openid 比对但 ARTIST_OPENID 来源改为 login 返回的 role 字段）

**isArtist 新逻辑决策**：
- 服务端 artist_profile._openid 是权威，但客户端无法直接查 artist_profile（无对应云函数 action）
- 方案：login 云函数返回 role 字段（当前 users.role 都是 'client'，artist 也如此）
- 折中：login 云函数增加逻辑——若 openid 匹配 artist_profile._openid 则 role='artist'（Plan 01 已让 artist_profile 可查）
- **本 task 不改 login 云函数**（避免跨 plan）：客户端 isArtist 暂时保留 openid 比对逻辑，但 ARTIST_OPENID 改为从 `wx.cloud.callFunction('login',{action:'getArtistOpenid'})` 动态获取（新增 action）
- **简化方案**：本 plan 直接在 login 云函数 getUser action 返回值中加 isArtist 布尔（查 artist_profile 比对 openid），客户端读 `_userInfo.isArtist`。这需要改 login/index.js（属于 Plan 01 范畴的扩展，但本 plan 调用）。

**最终采用简化方案**：
- 本 task 修改 `cloudfunctions/login/index.js`：getUser action 查询 artist_profile._openid，返回值增加 `is_artist: boolean`
- 客户端 `_userInfo.is_artist` 直接作为 isArtist 判据，移除 ARTIST_OPENID 比对
- 这让 SEC-04 完全闭环（客户端也摆脱 magic constant）

**Files updated (本 task 实际改动):**
- `miniprogram/services/auth.js`：增加 ensureLogin/refreshUserInfo，isArtist 改读 `_userInfo.is_artist`
- `cloudfunctions/login/index.js`：login action 和 getUser action 都增加 is_artist 字段（查 artist_profile）

**Verification:**
- `ensureLogin()` 重复调用返回同一 Promise（测试：const p1=ensureLogin(); const p2=ensureLogin(); p1===p2）
- `isArtist()` 在 ensureLogin resolve 后返回正确布尔
- login 云函数返回值含 is_artist 字段
