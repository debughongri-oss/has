**slider.js** — Component logic per D-06, D-07, D-09, D-10, D-11:

Properties (per D-10):
- `beforeSrc` (String, default `''`) — 妆前图 URL
- `afterSrc` (String, default `''`) — 妆后图 URL  
- `height` (String, default `'900rpx'`) — 容器高度

Data:
- `clipPercent` (Number, default `50`) — 滑块位置百分比

Methods:
- `onHandleTouchStart(e)` — Record initial touch X position and current clipPercent. Store in `this._touchStartX` and `this._startPercent`. Call `wx.createSelectorQuery()` to get the container width (`.ba-container`), cache in `this._containerWidth`.
- `onHandleTouchMove(e)` — Calculate deltaX = current X - start X. New percent = startPercent + (deltaX / containerWidth * 100). Clamp to [0, 100]. Call `setData({ clipPercent })` — use a throttle guard: only setData if the percent actually changed (compare with last value).
- `onHandleTouchEnd()` — No-op, cleanup if needed.
- `onFullscreenTap()` — Per D-11, trigger event `this.triggerEvent('fullscreen')`.

IMPORTANT implementation details (per D-06, D-07):
- Do NOT use `movable-view` component. Use manual touch events on the handle element.
- The touch events are bound on the `.ba-handle` element AND the `.ba-container` element (so dragging anywhere in the container moves the slider, not just on the tiny handle).

**slider.wxml** — Template per D-08:

```xml
<view class="ba-container" style="height: {{height}};">
  <!-- After image (bottom layer, full width) -->
  <image class="ba-image ba-image--after" src="{{afterSrc}}" mode="aspectFill" />

  <!-- Before image (overlay, clipped from right) per D-08 -->
  <view class="ba-before-clip" style="clip-path: inset(0 {{100 - clipPercent}}% 0 0);">
    <image class="ba-image ba-image--before" src="{{beforeSrc}}" mode="aspectFill" />
  </view>

  <!-- Divider line + handle per D-08 -->
  <view class="ba-divider" style="left: {{clipPercent}}%;">
    <view class="ba-handle">
      <view class="ba-handle-arrow ba-handle-arrow--left"></view>
      <view class="ba-handle-arrow ba-handle-arrow--right"></view>
    </view>
  </view>

  <!-- Labels per D-08 -->
  <view class="ba-label ba-label--before">BEFORE</view>
  <view class="ba-label ba-label--after">AFTER</view>

  <!-- Fullscreen button (top-right) -->
  <view class="ba-fullscreen tappable" bindtap="onFullscreenTap">
    <view class="ba-fullscreen-icon"></view>
  </view>

  <!-- Touch area covers entire container for easy dragging -->
  <view class="ba-touch-area"
    bindtouchstart="onHandleTouchStart"
    bindtouchmove="onHandleTouchMove"
    bindtouchend="onHandleTouchEnd">
  </view>
</view>
```

CRITICAL implementation notes:
1. The `ba-touch-area` sits above all other elements (z-index: 6) to capture all touch events. The fullscreen button must have `catchtap="onFullscreenTap"` instead of `bindtap` to prevent the touch area from consuming the tap. Actually, since the touch area uses touch events (not tap), bindtap on fullscreen button will work fine as taps and touch events don't conflict in mini programs.
2. The before image wrapper uses `clip-path: inset(0 {{100 - clipPercent}}% 0 0)` — this clips from the RIGHT side. When clipPercent=50, inset(0 50% 0 0) shows the left 50% (before). When clipPercent=100, inset(0 0% 0 0) shows full before image.
3. Use `catchtouchmove` on the touch area to prevent scroll interference — returning false or calling `e.preventDefault()` is NOT available in mini programs, but `catch` prefix prevents event propagation.
4. For the SelectorQuery in touchStart, query the `.ba-container` node's width. Cache it for the session.
