**Add `syncArtistRating()` helper (D-19 → RECOMPUTE strategy for concurrency safety):**
```js
// Recompute avg_rating/total_reviews from the reviews collection (idempotent, self-healing).
// Chosen over incremental math: WeChat cloud DB has weak transactions; incremental
// (old_avg*old_total ± new)/new_total suffers FP drift + races under concurrent writes.
// Review volume is tiny for a single artist → recompute cost negligible. Matches POL-06.
async function syncArtistRating() {
  const $ = db.command.aggregate
  const res = await db.collection('reviews').aggregate()
    .group({ _id:null, total: $.sum(1), avgRating: $.avg('$rating') }).end()
  const total = (res.list[0] && res.list[0].total) || 0
  const avg = (res.list[0] && res.list[0].avgRating) || 0
  // upsert artist_profile (single-doc collection)
  const existing = await db.collection('artist_profile').limit(1).get()
  if (existing.data.length) {
    await db.collection('artist_profile').doc(existing.data[0]._id)
      .update({ data:{ avg_rating: Number(avg.toFixed(2)), total_reviews: total, updated_at: db.serverDate() } })
  } else {
    await db.collection('artist_profile').add({ data:{ avg_rating: Number(avg.toFixed(2)), total_reviews: total } })
  }
}
```
Handle `isCollectionMissing` inside the helper (treat as total=0/avg=0).
